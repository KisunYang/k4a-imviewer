# k4a_imviewer 개발 보고서

## 1. 개요

### 1.1 목적

Microsoft **Azure Kinect DK**의 컬러·깊이 스트림을 실시간으로 표시하고, **사람 분할(RVM 등)** 과 **깊이 정렬**을 결합해 **바닥 영역에 접촉 그림자·캐스트 그림자·반사**를 합성하는 데스크톱 뷰어 애플리케이션을 개발하였다. 연구·데모·현장 검증에서 “그린 스크린 없이” 사람과 바닥의 관계를 시각화하는 것을 목표로 한다.

### 1.2 산출물

| 구분 | 내용 |
|------|------|
| 실행 파일 | `k4a_imviewer` (CMake 타깃명과 동일) |
| 핵심 소스 | `src/k4a_imviewer.cpp`, `src/rvm_floor_fx.cpp`, `src/rvm_floor_fx.hpp`, `src/viewer_settings_io.cpp` |
| 빌드 정의 | `CMakeLists.txt`, 배치 스크립트 (`build_k4a_imviewer.bat` 등) |
| 원격 저장소 | GitHub (`k4a-imviewer` 등 사용자 저장소에 연동 가능) |

## 2. 개발 환경 및 기술 스택

| 항목 | 선택 |
|------|------|
| 언어 | C++17 |
| 빌드 | CMake 3.20+, MSVC (Windows) |
| 디바이스 SDK | Azure Kinect Sensor SDK |
| 컴퓨터 비전 | OpenCV (core, imgproc, imgcodecs, dnn) |
| UI | Dear ImGui + GLFW + OpenGL 3 (Glad 로더) |
| 수학 | GLM (헤더 라이브러리, FetchContent) |
| 인물 분할 | Robust Video Matting(RVM) ONNX; ONNX Runtime(선택) 또는 OpenCV DNN 폴백 |

CMake는 **FetchContent**로 GLFW, ImGui(docking), GLM을 자동 가져오며, ONNX Runtime은 기본적으로 공식 릴리스 ZIP을 내려받아 `third_party/`에 풀도록 구성되어 있다(저장소에는 대용량 바이너리를 포함하지 않는 것을 권장).

## 3. 시스템 구성

### 3.1 데이터 흐름

1. Kinect에서 **Depth / Color** 캡처(및 선택적으로 IR).
2. 컬러와 깊이를 SDK **변환 파이프라인**으로 정렬(`color_depth_aligned_mats` 등).
3. **RVM ONNX** 또는 **깊이 기반 폴백**으로 사람 알파 맵(CV_32F, 0~1) 생성.
4. 깊이·알파로 **바닥 평면(RANSAC)** 및 **바닥 마스크** 추정.
5. 바닥 마스크 위에 **접촉 그림자 → 거울형 바닥 반사 → 3D 캐스트 그림자(평행광)** 순으로 작업 버퍼에 합성 후, 알파로 원본 사람과 블렌딩.

### 3.2 모듈 역할

- **`k4a_imviewer.cpp`**: 장치 열기/스트림, ImGui 패널, 깊이 컬러맵, MR·플로어 패널, `rvm_floor::Params` 구성 및 합성 호출.
- **`rvm_floor_fx.cpp` / `rvm_floor_fx.hpp`**: 바닥 평면·마스크, 그림자·반사·RVM 추론, `compose_floor_with_fx`, `compute_cast_shadow_weight` 등 핵심 알고리즘.
- **`viewer_settings_io.cpp`**: UI 설정과 INI 간 저장/로드.

## 4. 주요 기능 요약

### 4.1 카메라·뷰

- 컬러/깊이/IR 표시, 해상도·FPS·포맷 선택.
- 깊이를 컬러 해상도에 맞춘 오버레이·MR 관련 경로.

### 4.2 바닥 합성 (`rvm_floor`)

- **바닥 평면**: 카메라 공간 3점 RANSAC; 발 근처 ROI 옵션; 프레임 간 EMA로 평면 안정화.
- **바닥 마스크**: 평면 거리 임계 + 형태학·연결요소로 하부 큰 영역 유지; 실패 시 화면 하단 사각형 폴백.
- **접촉 그림자**: 사람 무게중심·발 추정 기반 가우시안 감쇠.
- **반사**: 수평 거울면(`reflection_plane_y`) 대칭 색 블렌딩.
- **3D 캐스트 그림자**: 태양 고도·방위로 방향 벡터 **L** 정의; 사람 픽셀을 깊이로 3D화 후 광선과 바닥 평면 교차점을 이미지에 스플랫; 블러 및 최댓값 정규화 후 `cast_shadow_alpha`로 어둡게 합성.

### 4.3 RVM

- ONNX 경로 지정 후 로드; ONNX Runtime 우선, 실패 시 OpenCV `readNetFromONNX`.
- 추론 결과 알파를 입력 해상도에 리사이즈하여 후속 합성에 사용.

### 4.4 설정

- INI 기반 설정 저장(`viewer_settings_io`), 바닥·태양·그림자·평면 파라미터 유지.

## 5. 코딩·품질

- **`rvm_floor_fx`**: 주요 함수·자료구조에 **한글 주석**을 추가하여 평면 추정·그림자 파이프라인 이해를 돕도록 정리하였다.
- **Git**: 저장소 루트에 `.gitignore`로 `build/`, 로컬 OpenCV 트리, 대형 ONNX 가중치, ONNX Runtime 바이너리 등을 제외하여 GitHub 업로드 용량과 보안을 관리한다.

## 6. 빌드·실행

```powershell
cmake -S . -B build -DOpenCV_DIR="<OpenCVConfig.cmake 경로>"
cmake --build build --config Release
```

- **K4A_ROOT**: Kinect SDK 설치 경로(기본값은 CMake에 예시 경로로 설정).
- **OpenCV_DIR**: 빌드 산출물의 `OpenCVConfig.cmake` 위치.

실행 시 Kinect 연결 및 드라이버 정상 여부가 필요하다.

## 8. 개발 과정에서의 기술적 판단 요약

개발 중 다음과 같은 시도·조정이 있었다.

- **그림자 일관성**: 캐스트 맵 정규화·후처리·곡선 등 여러 방향을 검토하였으며, 최종적으로는 유지보수와 동작 안정성을 우선하여 상대적으로 단순한 파이프라인으로 정리한 구간이 있다.
- **Kinect 동기 큐**: 처리 지연 시 SDK `capturesync` 경고가 발생할 수 있어, 프레임 폴링 방식과 부하 분산(예: 정렬 스트라이드)이 중요함을 확인하였다.
- **저장소 구조**: ONNX/OpenCV 바이너리는 저장소에 넣지 않고 CMake·문서로 획득 경로를 안내하는 편이 유지보수에 유리하다.

(상세 커밋 단위 이력은 Git 로그를 참조한다.)
